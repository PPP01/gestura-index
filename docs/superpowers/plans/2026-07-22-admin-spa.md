# Admin-SPA (SP4b) — Implementierungsplan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ein client-only Svelte-5-Admin-Panel im bestehenden Frontend, das die `/api/admin`-API (SP4a) in voller Parität bedienbar macht: WebAuthn-Login + reaktiver Step-up, eigene Passkey-Verwaltung, komplette Moderation, Nutzerverwaltung und Audit-Log.

**Architecture:** Gleiche SvelteKit-App. Neue präfixlose Route-Gruppe `src/routes/admin/` (Locale-Präfix kommt wie überall über den `reroute`-Hook). Ein `admin/+layout@.svelte` bricht aus dem öffentlichen Root-Layout aus (eigenes Sidebar-Chrome ohne Header/Footer) und `admin/+layout.ts` setzt `prerender=false; ssr=false` (nutzt den bereits konfigurierten `adapter-static`-`fallback: '200.html'`). Ein typisierter Admin-API-Client (credentials + `X-Requested-With`, `AdminApiError` mit `stepUpRequired`/`backupRequired`-Flags) spiegelt das bestehende `src/lib/api.ts`-Muster. WebAuthn-Ceremonies über `@simplewebauthn/browser`; in Tests gemockt.

**Tech Stack:** Svelte 5 (Runes), TypeScript (strict), `@sveltejs/adapter-static`, `@inlang/paraglide-js` (en/de), `@lucide/svelte`, `@simplewebauthn/browser` (neu), Vitest + `@testing-library/svelte` + jsdom.

**Spec:** `docs/superpowers/specs/2026-07-22-admin-spa-design.md` — bei Detailfragen gilt der Spec.

## Verifizierte API-Verträge (SP4a, live unter `/api/admin`)

- Auth: `POST /auth/options` → RequestOptions-JSON; `POST /auth/login` (Assertion) → 204 + Session-Cookie; `POST /auth/logout` → 204; `GET /auth/me` → `{ email, displayName, role: 'admin'|'moderator', credentialCount, stepUpFresh }`.
- Register: `POST /register/options` (`{token}`) → CreationOptions-JSON; `POST /register` (`{token, …attestation}`) → 201 `{status:'active'}`.
- Step-up: `POST /stepup/options` → RequestOptions-JSON; `POST /stepup` (Assertion) → 204.
- Credentials: `GET /credentials` → `[{id,label,createdAt,lastUsedAt}]`; `POST /credentials/options` → CreationOptions; `POST /credentials` (`{…attestation, label}`) → 201 `{id,label}`; `PATCH /credentials/{id}` (`{label}`) → `{id,label}`; `POST /credentials/{id}/remove` → 204 (409 `{backupRequired}` wenn danach <2; Step-up-geschützt).
- Moderation: `GET /queue` → `{entries:[{id,formatId,type,createdAt}], versions:[{id,entryId,semver,hasTransform}]}`; `GET /entries/{id}` → Detail (EntrySerializer::toDetail) + `reports` **+ `submitterId` (int) + `submitterBanned` (bool)** (ergänzt in der SP4a-Vorarbeit, Commit `5acdc97`); `POST /entries/{id}/approve|reject`; `POST /versions/{id}/approve|reject`; `GET /reports` → offene Meldungen, **jedes Item mit `submitterId` + `submitterBanned`**; `POST /reports/{id}/resolve` (`{publish:bool}`); `POST /submitters/{id}/ban|unban`. **Kein `GET /submitters`-Listen-Endpunkt** — Ban/Unban erfolgen kontextuell über `submitterId` aus Eintrag-Detail/Meldung.
- Users (admin): `GET /users`; `POST /users` (`{displayName,email,role}`) → 201; `POST /users/{id}/disable|enable|reinvite`.
- Audit (admin): `GET /audit?page&perPage` → `{items:[{id,actor,action,targetType,targetId,detail,createdAt}], page, perPage}`.
- Fehler: `application/problem+json` mit `title`/`detail`; Extra-Flags `stepUpRequired` (403), `backupRequired` (409), `retryAfter` (429). Destruktive Aktionen (reject Eintrag/Version, ban, invite, disable, enable, credential remove) verlangen Step-up (403) und teils Backup-Gate (409).

```ts
import { env } from '$env/dynamic/public';

export const ADMIN_API_BASE = env.PUBLIC_API_BASE || 'https://api.gestura.eu';

export class AdminApiError extends Error {
  constructor(
    public status: number,
    public title: string,
    public detail: string | null,
    public stepUpRequired = false,
    public backupRequired = false,
    public retryAfter: number | null = null,
  ) { super(title); this.name = 'AdminApiError'; }
}

export interface AdminClientOpts { fetch?: typeof fetch; baseUrl?: string }

async function toError(res: Response): Promise<AdminApiError> {
  let title = res.statusText || 'Error', detail: string | null = null, su = false, bu = false, ra: number | null = null;
  try {
    const b = await res.json();
    if (typeof b?.title === 'string') title = b.title;
    if (typeof b?.detail === 'string') detail = b.detail;
    su = b?.stepUpRequired === true; bu = b?.backupRequired === true;
    ra = typeof b?.retryAfter === 'number' ? b.retryAfter : null;
  } catch { /* kein JSON-Body */ }
  return new AdminApiError(res.status, title, detail, su, bu, ra);
}

export async function adminFetch<T>(path: string, init: RequestInit = {}, opts: AdminClientOpts = {}): Promise<T> {
  const f = opts.fetch ?? fetch;
  const base = opts.baseUrl ?? ADMIN_API_BASE;
  let res: Response;
  try {
    res = await f(`${base}${path}`, {
      ...init,
      credentials: 'include',
      headers: { 'Content-Type': 'application/json', 'X-Requested-With': 'XMLHttpRequest', ...(init.headers ?? {}) },
    });
  } catch {
    throw new AdminApiError(0, 'Network error', null);
  }
  if (!res.ok) throw await toError(res);
  if (res.status === 204) return undefined as T;
  const text = await res.text();
  return (text ? JSON.parse(text) : undefined) as T;
}
```

import { me, AdminApiError, type MeResponse, type AdminClientOpts } from './api';

class Session {
  user = $state<MeResponse | null>(null);
  get isAdmin() { return this.user?.role === 'admin'; }
  get needsBackup() { return (this.user?.credentialCount ?? 0) < 2; }
  set(u: MeResponse | null) { this.user = u; }
  clear() { this.user = null; }
  async load(opts?: AdminClientOpts): Promise<MeResponse | null> {
    try { this.user = await me(opts); return this.user; }
    catch (e) { if (e instanceof AdminApiError && e.status === 401) { this.user = null; return null; } throw e; }
  }
}
export const session = new Session();

```ts
import { AdminApiError, stepUpOptions, stepUpVerify, type AdminClientOpts } from './api';
import { performAssertion } from './webauthn';

export async function withStepUp<T>(action: () => Promise<T>, opts?: AdminClientOpts): Promise<T> {
  try { return await action(); }
  catch (e) {
    if (!(e instanceof AdminApiError) || !e.stepUpRequired) throw e;
    const options = await stepUpOptions(opts);
    const assertion = await performAssertion(options);
    await stepUpVerify(assertion, opts);
    return action(); // einmaliger Retry
  }
}
```
